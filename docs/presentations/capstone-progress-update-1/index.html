The Friction

Getting dressed is a daily decision-fatigue problem

Most people own more clothes than they actually wear. The paradox of choice leads to the same 5 outfits on repeat — while the rest of the wardrobe sits untouched. Existing apps either require manual outfit curation or push you toward buying more.

What is OutfAI?

03 / 12
P-01

Wardrobe-First

Your closet is the system of record. Every recommendation starts from garments you actually own — not products a retailer wants to sell you.

P-02

Context-Aware

Outfits are generated with real context: your mood right now, today's weather and temperature, and the occasion — work, casual, social, formal, athletic.

P-03

Explainable by Design

A rule-based scoring engine (not a black-box ML model) produces a 0–100 score with a transparent breakdown: color harmony, style coherence, mood alignment, occasion fit, and versatility.

P-04

Commerce as an Option, Not the Goal

Purchase suggestions are offered only when a gap in the wardrobe is identified — and any suggestion must justify how it integrates with what you already own.

06

Recommendation Engine

07 / 12
1

Context Filter

Filters garments by weather suitability, temperature range, season, and occasion. Ensures only appropriate items reach candidate generation.

2

Candidate Generation

Generates all valid combinations: tops × bottoms × shoes, optionally layered with outerwear and accessories. Applies hard constraints (e.g. no shorts in winter).

3

Score & Rank

Scores each candidate 0–100 using 6 weighted factors. Candidates below 60 are discarded. Top 6 results are returned, diversity-boosted to avoid repetition.

Scoring Breakdown (max 100 pts)
Base score
50 pts
Color harmony
+20 pts

+20 pts

+5–10

API Layer — tRPC

08 / 12
Implemented Endpoints
QUERY
recommendations.generate
Generate outfit recommendations from closet + context. Returns scored & ranked outfits.
QUERY
recommendations.getOutfit
Fetch a single outfit by ID with full score breakdown.
MUTATION
recommendations.logInteraction
Log user feedback (save / skip / wear) for future learning.
All inputs validated with Zod. Type-safe end-to-end — no API contract drift.
// recommendations.generate input schema const GenerateInput = z.object({ mood: z.enum(['energetic', 'calm', 'bold', ...]), occasion: z.enum(['casual', 'work', 'formal', ...]), weather: z.object({ temp: z.number(), condition: z.string(), season: z.enum(['spring', 'summer', ...]), }), limit: z.number().optional().default(6), }); // Output: scored outfit candidates interface OutfitRecommendation { id: string; items: Garment[]; score: number; // 0–100 breakdown: ScoreBreakdown; explanation: string; occasion: Occasion; }
